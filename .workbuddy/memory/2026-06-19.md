# 2026-06-19 — Flutter Bluetooth Plugin Project Created

## What was built
Created a complete Flutter Bluetooth plugin (`flutter_bluetooth`) at `E:\Android\AndroidStudioProjects\flutter_bluetooth\`.

## Architecture
- **API**: mimics `flutter_blue_plus` for familiarity
- **Platform**: Android-only (API 21+)
- **Scanning**: Supports both classic Bluetooth (BR/EDR) discovery and BLE scanning simultaneously
- **BLE**: Full GATT connection, service discovery, read/write/notify, MTU, bonding

## Key files
- `lib/src/flutter_bluetooth_impl.dart` — Main `FlutterBluetooth` class (singleton)
- `lib/src/bluetooth_device.dart` — `BluetoothDevice` with connect/disconnect/GATT ops
- `lib/src/bluetooth_characteristic.dart` — GATT characteristic read/write/notify
- `lib/src/bluetooth_service.dart` — GATT service model
- `lib/src/bluetooth_descriptor.dart` — GATT descriptor read/write
- `lib/src/utils.dart` — `Guid`, `ScanResult`, `AdvertisementData`, enums
- `android/.../FlutterBluetoothPlugin.kt` — MethodChannel handler (18+ methods)
- `android/.../BleManager.kt` — BLE scan + GATT connection management
- `android/.../ClassicBluetoothManager.kt` — Classic BT discovery via BroadcastReceiver
- `example/lib/main.dart` — Demo app with scan/connect UI

## Communication
- MethodChannel: `io.github.berial.flutter_bluetooth/methods`
- EventChannel: `io.github.berial.flutter_bluetooth/events` (async events: scan results, connection state, adapter state, notifications)

## 2026-06-19 — Added Data Communication (Send + Continuous Receive)

### What was added
Connection成功后，设备现在支持**发送数据**和**持续接收数据**：

#### Classic Bluetooth (RFCOMM SPP)
- New file: `android/.../RfcommManager.kt` — RFCOMM 套接字连接管理
  - `connect()` — 建立 RFCOMM SPP 连接 (默认 UUID: 00001101-...)
  - `sendData()` — 通过 OutputStream 发送字节数据
  - 后台协程持续读取 InputStream，自动发送 `rfcommDataReceived` 事件
  - 支持安全/不安全两种连接方式的 fallback
- `BluetoothDevice` 新增方法:
  - `connectRfcomm()` — RFCOMM 连接
  - `sendRfcommData(List<int>)` — 发送数据
  - `onRfcommDataReceived` — Stream 持续接收
  - `disconnectRfcomm()` — 关闭连接
  - `checkRfcommConnected()` — 检查状态

#### BLE (GATT)
- `BluetoothDevice` 新增便捷方法:
  - `sendBleData(value, serviceUuid, characteristicUuid)` — 发送到指定特征
  - `getBleDataStream(serviceUuid, characteristicUuid)` — 获取数据流 (自动 discoverServices + setNotifyValue)

#### Example App
- 新增 `DataCommunicationPage` — 完整的数据通信界面
  - 发送文本框 + Send 按钮
  - 实时数据日志 (SEND/RECV，支持 UTF-8 文本和十六进制显示)
  - BLE 模式下可选择特征进行通知订阅

## 2026-06-19 — 修复私有方法跨文件访问问题

### 问题
`bluetooth_characteristic.dart` 等文件调用 `FlutterBluetooth._readCharacteristic()` 等 `_` 前缀私有方法。
Dart 中 `_` 是 library-private，不同 `.dart` 文件是不同的 library，无法跨文件访问私有成员。

### 解决
用 `part` / `part of` 将 6 个 `lib/src/*.dart` 文件合并为同一个 library：
- `flutter_bluetooth.dart` → `library flutter_bluetooth;` + 6 个 `part 'src/xxx.dart';`
- 每个 `src/xxx.dart` → 首行 `part of '../flutter_bluetooth.dart';`
- 移除所有 src 内部的跨文件 import（共 10 处），因为同一 library 内无需 import
